# 报告风格主题提示词

## 主题

不同报告风格的差异化生成要求。

## 使用位置

`build_prompt(...)` 中的 `{style_instruction}`。运行时代码会从 `prompts/report-styles/` 读取对应风格文件。

## 风格主题

| 报告风格 | 风格要求 |
| --- | --- |
| 科研日报 | 突出最新进展、关键发现、研究问题和代表性方法，适合日常快速跟踪。 |
| 综述摘要 | 突出领域背景、研究主线、共识与分歧，弱化新闻式表达，形成更像小型综述的摘要。 |
| 技术路线图 | 突出材料体系、工艺路线、表征方法、性能目标之间的路径关系，并指出可延伸路线。 |
| 专利机会 | 突出可保护的技术组合、应用场景、差异化改进点、潜在规避空间和后续专利挖掘机会，但不得虚构未在摘要中出现的效果。 |
| 研究建议 | 突出可执行研究假设、变量设计、对照设置、表征手段和风险点，建议必须来源于文献已给出的材料/方法/问题。 |

## 使用建议

- `科研日报` 适合高频跟踪和快速阅读。
- `综述摘要` 适合阶段性汇报、开题背景和领域概览。
- `技术路线图` 适合梳理工艺、材料体系和性能目标之间的关系。
- `专利机会` 适合专利挖掘前的技术组合归纳，但仍需人工判断新颖性和创造性。
- `研究建议` 适合从已有文献中提炼下一步研究变量和验证路径。
